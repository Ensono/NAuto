Configuration

You can configure NAuto.AutoBuild either inline as part of the fluent interface or by passing in an instance of AutoBuilderCOnfiguration.

Shows an AutoBuilderConfiguration class being passed into auto build. 
var config = new AutoBuilderConfiguration(
            defaultLanguage: Language.Chinese);

var myModel = NAuto.AutoBuild<MyModel>(config)
    .Construct()
    .Build();
Shows an example of inline configuration. 
var myModel = NAuto.AutoBuild<MyModel>()
            .Configure(x => x.DefaultLanguage = Language.Russian)
            .Construct()
            .Build();

It is also perfectly valid to pass in an instance of AutoBuilderConfiguration and still use inline .Configure calls to override certain properties.

Conventions

Conventions can be configured inline or via the AutoBuilderConfiguration instance.

Typically it is a good idea to create a configuration class for a particular project/domain area which is shared between tests.

This shared configuration file would also have a set of conventions specific to the area.

The following is an example of this approach:

var myModel = NAuto.AutoBuild(MyConfig.GetConfig())
    .Construct()
    .Build();
       
public static class MyConfig
{
    public static AutoBuilderConfiguration GetConfig()
    {
        var autoBuilderConfiguration = new AutoBuilderConfiguration();
        autoBuilderConfiguration.DefaultLanguage = Language.Chinese;

        autoBuilderConfiguration.Conventions.Add(
            new ConventionMap(
                ConventionFilterType.Contains,
                "Foo",
                typeof(string),
                config => "Bar"));

        return autoBuilderConfiguration;
    }
}
