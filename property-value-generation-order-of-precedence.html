Property value generation - order of precedence

Properties are generated according to a set order of precedence.

It is important to understand this as it may help diagnose an issue if the value generated does not match the expected.

1. Check if property already has value

This scenario happens when a constructor sets a class property.

In this case the set value is left unchanged.

2. Check if convention exists for property name and type

If a convention has been registered which matches the propety name and type, the convention result function is actioned with the value set to the property.

3. Check if property has DataAnnotation condition that must be adhered to

If a DataAnnotation has been added to the property, a random value for the specified type is created, it will however adhere to the conditions specified in the property attribute rather than use the value from the AutoBuilderConfiguration settings.

For example if the property is decorated with a [Range(10, 15] attribute, the resulting integer will be between 10 and 15.

4. Generate random value based on AutoBuilderconfiguration settings

If this point is reached, a random property value we be set using the AutoBuilderConfiguration settings as a guide.
