Supported Types

NAuto will build most models/objects including custom objects.

Eventually most objects get down to built in .net types such as primitives, strings and arrays etc.

This page is here to document specific types we have tested.

Just to be clear, types can be complex and have complex properties, the following list is made up of built in types that are supported for use in your models.

  • string
  • int
  • int?
  • double
  • double?
  • decimal
  • decimal?
  • bool
  • bool?
  • long
  • long?
  • char
  • char?
  • Guid
  • byte
  • byte?
  • DateTime
  • Enums
  • Array
  • List<>
  • Dictionary<,>

So for example the following model would be perfectly fine:

public class ComplexType
{
    public SubComplexType MySubComplexType { get; set; }
    public Dictionary<int, SubComplexType> MyDictionary { get; set; }
    public int MyInt { get; set; }
    public bool IsABool { get; set; }
    public List<string> SomeStrings { get; set; }
}

public class SubComplexType
{
    public decimal? MyNullableDecimal { get; set; }
}
