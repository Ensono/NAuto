Conventions

Before any property value is generated within an object graph, the conventions api is called to see if a convention exists and should be used instead.

Adding a convention can be done in a few different ways:

  • Added to Conventions list on an instance of AutoBuilderConfiguration
  • Added directly to AutoBuild method before Construct is called.

For a more detailed explanation of how to configure NAuto and conventions, please refer to the configuration section.

Conventions require the following data:

ConventionFilterType

This enumeration tells NAuto how to check if the convention filter string matches a property name

The enumeration options are:

  • StartsWith
  • EndsWith
  • Contains
  • Equals

ConventionFilter

The convention filter is just a simple string which is used to match against a property name.

Type

Type is literally the property type the convention will match with.

For example, if you create a convention for generating datetimes, you would pass typeof(DateTime).

Result

Result requires a Func which is to be called when a convention condition is matched.

The Func must accept a parameter of AutoBuilderConfiguration and return the created value.

Example

The following example shows a convention that states for any string that starts with Foo, Bar should be returned:

var myModel = NAuto.AutoBuild<MyModel>()
    .AddConvention(
            ConventionFilterType.StartsWith,
            "Foo",
            typeof(string),
            config => "Bar")
    .Construct()
    .Build();
