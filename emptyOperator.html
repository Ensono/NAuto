Using the Empty operator

As well as auto generating model data, NAuto allows you to use the same syntax to create an instance of an object and specify only the values you want to set. This allows you to use a consistent approach for generating test data for your objects in tests, even when you have no need to auto generate test data.

The Empty operator is simply used instead of the Construct operator to prevent any test data being auto generated. Take the following example:

If we had the a very simple object as below

public class User
{
    public string Name { get; set; }

    public int Age { get; set; }

    public string Email { get; set; }

    public string Password { get; set; }
}

If we simply wanted to initialise a new object and set only the email address and password, leaving the name and age property unset, you would do the following:

var user = NAuto.AutoBuild()
            .Empty()
            .With(x => x.Email = "a@b.com")
            .With(x => x.Password = "pa55word")
            .Build();

This results in Email being set to "a@b.com", Password set to "pa55word" with Name set to the default string.empty and Age set to the default 0.
