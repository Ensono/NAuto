Data Annotations support in NAuto

NAuto supports System.ComponentModel.DataAnnotations, allowing Data Annotations applied to properties to be read and used when auto generating test data. This provides a nice easy way to ensure that property values generated to conform to the validation placed on properties through the use of Data Annotations.

Currently the following Data Annotation attributes are supported:

  • StringLength
  • Range

Take the following example object with data annotations included:

public class User
{
        [StringLength(20)]
        public string Name { get; set; }

        [Range(10, 20)]
        public int MyRange { get; set; }
}

If we were to run the standard AutoBuild command as follows:

var user = NAuto.AutoBuild<User>()
            .Construct()
            .Build();

The test data values that are generated will conform to the lengths specified within the data annotation, without doing anything different in the AutoBuild command

Of course any property that has a data annotation specified can have its value overridden using the standard With operator.

The following example overrides the data annotations on the Age property by specifying that the value must be between 5 and 50

var user = NAuto.AutoBuild<User>()
             .Construct()
             .With(x => x.Age, 5, 50)
             .Build();
