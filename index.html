What is it?

NAuto is a .Net library for generating test data

Why use it?

Multi language support and convention based data generation makes it a very powerful tool for both unit testing and generating integration test data.

It's very easy to use and in most cases very little code needs to be written.

Instead of generating unstructured nonsense data, NAutos conventions and property types ensure data is well structured and semi-realistic.

Happily works with NUnit and MS Test and will work fine with any mocking container such as Moq or Rhino Mocks.

What can it do?

  • Create large complex object graphs with hardly any code
  • Create realistic data using extendable conventions
  • Create Multi-Language data i.e Chinese/Russian/German/etc.
  • Randomization Helpers for:
    • Lists
    • Strings
    • Integers
    • Doubles
    • etc.
  • Random property type generation for:
    • Emails
    • Urls
    • Postal Codes
    • Phone Numbers
    • etc.
  • Built in optional persistence for repeatable test data
  • Respects .NET/MVC data annotations such as StringLength/Range/MinLength/MaxLength etc.
  • And a load of other features that you can find out about throughout this site

What does it look like?

Generating a model with the Id property overridden 
var productViewModel = NAuto.AutoBuild<ProductViewModel>()
    .Construct()
    .With(x => x.Id = 1234)
    .Build();
Overriding a property when the specified condition is met 
var model = NAuto.AutoBuild<ProductViewModel>()
    .Construct()
    .If(x => x.Stock == 0)
    .Then(x => x.OutOfStock = true)
    .Build();
Generating a valid email address format using the Russian character set 
var email = NAuto.GetRandomPropertyType(
              PropertyType.Email,
              Language.Russian);
Generating a alphanumeric string with a specified length 
var productCode = NAuto.GetRandomString(
              length,
              CharacterSetType.AlphaNumeric);
